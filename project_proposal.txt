Project Proposal (5%)
Title: Instagram OSINT Knowledge Graph Agent (Pure GraphRAG)

1) Project Summary
This project builds a six-stage pipeline that transforms publicly available Instagram-style content into a provenance-aware knowledge graph. The final goal is to support natural-language questions answered through graph traversal (Pure GraphRAG), with evidence linked back to source artifacts.

2) Motivation
Traditional keyword or embedding-only retrieval often misses multi-hop relationships (for example, "who appeared together most often"). A graph-first approach better captures entity connections and supports traceable, explainable answers.

3) Objectives
- Ingest public post-like data in a compliant way.
- Extract entities and relations from text content.
- Deduplicate mentions into canonical entities.
- Insert graph-ready nodes/edges with provenance.
- Enable read-only graph retrieval for grounded QA.

4) Scope and Constraints
- Public-data-only and compliance-first.
- No private accounts, DMs, or bypass behavior.
- Pure GraphRAG direction for query-time retrieval.
- Embeddings are used only for dedup assistance (not final query retrieval).

5) System Design (6 Stages)
Stage 1: Collection
- Input: fixture/live public feed
- Output: RawArtifact records in SQLite

Stage 2: Extraction
- Input: raw artifacts
- Output: ExtractionRecord (entities, relations, confidence)

Stage 3: Deduplication
- Input: extraction records
- Output: DedupReport (canonical ids, audit log)

Stage 4: Graph Insertion
- Input: dedup + extraction + artifacts
- Output: graph nodes/relationships with provenance

Stage 5: Quality Checking
- Input: graph snapshot
- Output: quality report (anomalies/conflicts/low confidence)

Stage 6: Query Answering
- Input: natural-language question
- Output: graph-grounded answer with evidence and provenance

6) Tech Stack
- Python 3.10+
- SQLite (raw/extraction/dedup/graph-lite)
- RapidFuzz for fuzzy matching
- Pydantic schemas and CLI entrypoints
- Optional Neo4j backend planned for graph phase expansion

7) Current Progress
- Phase 1 complete (collection)
- Phase 2 complete (extraction)
- Phase 3 complete (deduplication)
- Phase 4 implemented (graph insertion, idempotent upserts, provenance fields)
- Phases 5-6 next

8) Deliverables
- Working pipeline commands: collect, extract, dedup, graph_insert
- Tests and lint passing
- Architecture documentation + phase docs
- Demo showing graph-ready evidence chain from source to relation

9) Evaluation Plan
- Functional: end-to-end fixture run through graph insertion
- Reliability: rerun idempotency (stable graph counts)
- Traceability: provenance retrievable on inserted relationships
- Quality: flagged ambiguous/low-confidence cases

10) Expected Impact
The project demonstrates a practical, explainable GraphRAG-style pipeline for OSINT-style reasoning and can serve as a foundation for advanced QA and graph analytics.
