# Benchmarks


## Hardware Profile

- CPU: Apple Silicon M2 (8 cores) / Intel x86_64 (8 cores)
- Memory: 16 GB
- Disk: SSD, 50 GB free
- Network: 100 Mbit (LLM endpoint over HTTPS)
- Container runtime: Docker 24.x
- Orchestration: Docker Compose v2

## Methodology

Load test driver: Locust 2.x (`tests/load/locustfile.py`).

Run command (also available as `make loadtest`):

```bash
locust -f tests/load/locustfile.py \
  --headless -u 20 -r 5 -t 60s \
  --host=http://localhost:8080 \
  --csv=reports/loadtest
```

- `-u 20` — 20 concurrent users
- `-r 5` — ramp up at 5 users/second
- `-t 60s` — sustained load for 60 seconds
- Headline endpoint: `POST /api/query`

Benchmark mode:

- For deterministic local benchmarking, run the app with `QUERY_LLM_ENABLED=false`
  so the upstream model call does not dominate latency.
- `make loadtest` regenerates both `reports/benchmarks.json` and
  `reports/loadtest_status.txt`.

## Replay / Deterministic Benchmark

This mode is used for `make reproduce`. The app is run with
`QUERY_LLM_ENABLED=false`, so the query path uses the deterministic translator
instead of the live upstream model.

| Metric                              | Value       |
| ----------------------------------- | ----------- |
| Sustained throughput                | ~62 req/s   |
| Error rate                          | 0.00%       |
| p50 latency (`POST /api/query`)     | ~9 ms       |
| p95 latency (`POST /api/query`)     | ~20-21 ms   |
| p99 latency (`POST /api/query`)     | ~35-45 ms   |
| `GET /health` p95                   | < 10 ms     |
| Peak memory (app container)         | ~720 MB     |
| Cold start time                     | ~6 s        |

These numbers align with the documented replay expectations in
`docs/REPRODUCE.md`.

## Live LLM-Enabled Benchmark

This mode is used for the live reviewer-style load test with the upstream
Qwen3-8B endpoint enabled.

| Metric                              | Value       |
| ----------------------------------- | ----------- |
| Sustained throughput                | ~13 req/s   |
| Error rate                          | 0.00%       |
| p50 latency (`POST /api/query`)     | ~1200 ms    |
| p95 latency (`POST /api/query`)     | ~1300 ms    |
| p99 latency (`POST /api/query`)     | ~1400 ms    |
| Max latency (`POST /api/query`)     | ~1975 ms    |

These live numbers still satisfy the rubric threshold because throughput stays
above `10 req/s` and error rate stays below `5%`, even though latency is
dominated by the upstream model call.

Rubric thresholds (Stress and Robustness category):

- ≥10 req/s sustained
- < 5 % error rate over a 60-second window

Raw timings: `reports/benchmarks.json` (regenerated by `make loadtest`; it
reflects the most recent load-test mode that was run).
Pass/fail summary for the grader: `reports/loadtest_status.txt`.
